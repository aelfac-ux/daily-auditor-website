AI × AUDIT · DAY 03

Let Cloud AI Write the Code—and Keep Company Data Local

Let AI see the method, not the secrets. Draft the program in the cloud; run confidential data, anomaly analysis, and evidence generation on the company machine.

Arthur Agent Editorial DeskJune 28, 20268-minute read
FIELD CASEAn auditor must test 120,000 payments but cannot upload supplier names, bank accounts, or amounts. She shares only generic field names, data types, and business rules with AI, then runs the resulting Python script locally.

The output identifies 86 high-risk transactions and preserves a complete run log. The original data never leave the machine.

The useful AI superpower is not dropping company data into a chat window. It is borrowing AI's coding ability while keeping a firm grip on the data boundary.

Separate method from data

Describe the schema without real values: rename fields C01 and C02, explain types and allowed patterns, and use fabricated examples. Ask AI to produce a script that reads a local file and writes results locally. Review the code before it sees any real record.

A seven-step workflow

  1. Define the audit objective and anomaly rules.
  2. Classify the data and confirm approved tools.
  3. Replace sensitive field names with generic labels.
  4. Ask AI to draft code using synthetic examples only.
  5. Review libraries, file access, network calls, and dangerous functions.
  6. Run the script in an approved local environment.
  7. Validate samples and retain code version, parameters, counts, and logs.

AI may write the net. Auditors still decide where to cast it—and whether the catch is evidence or seaweed.

What to inspect before execution

  • Unexpected network libraries or external API calls.
  • Use of eval, exec, shell commands, or package installation.
  • Hard-coded paths, credentials, or uncontrolled output locations.
  • Rules that silently drop nulls, duplicates, or failed conversions.
  • Missing logs for input counts, exceptions, and generated files.

Excel and TXT are enough to begin

Write suspicious records to a review workbook and execution details to a plain-text log. The point is not a fancy platform; it is a reproducible trail from population to exception to human conclusion.

CASE RESOLVED

From 120,000 payments to 40 reviewable samples

Local Python applies six rules, ranks 1,430 exceptions, narrows them to 86 high-risk items, then selects 40 representative samples. Auditors inspect contracts, receiving evidence, and approvals. The conclusion comes from evidence—not from AI's confidence.
