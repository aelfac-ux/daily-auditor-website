AI × AUDIT · DAY 06

How Auditors Can Save 50% of Their Time with AI

A case about the hours that went missing—and how to reclaim them without handing professional judgment to a machine wearing a very confident hat.

Arthur Agent Editorial DeskJuly 1, 20269-minute read
FIELD CASE · THE MISSING HOURSA manufacturing audit team must sample SAP procurement transactions, match purchase orders, receipts, and invoices, then draft the exceptions. Five days later, its most experienced auditor is still cleaning columns, opening PDFs, and polishing sentences.

“Where did professional judgment go?” asks the captain. It is waiting for Excel to finish pasting.

The best use of AI is not to reduce thinking time. It is to reduce the repetitive work that leaves auditors no time to think.

Secure the scene: 50% is not a promise

Processes, data quality, and review requirements differ. Treat “save 50%” as an improvement hypothesis: measure the baseline, run a controlled pilot, and compare total effort, errors, and rework. Counting only machine runtime while ignoring setup and review is how an efficiency report becomes its own audit finding.

THE EVIDENCE FORMULA
Net time saved =
(baseline effort - new total effort) / baseline effort

New total effort =
setup + execution + exception handling
+ human review + rework

Three suspects in the disappearance

  1. Data preparation: exporting SAP data, correcting formats, cleaning fields, stratifying, and sampling.
  2. Document matching: opening POs, goods receipts, and invoices to capture dates, values, and identifiers.
  3. Draft production: turning evidence into condition, criteria, cause, impact, and recommendation.

These tasks are not judgment-free, but large parts can be expressed as rules. The practical move is not “automate the audit.” It is to isolate repeatable, testable, reversible tasks.

Case 1: SAP sampling—AI writes the net, Python casts it

The auditor defines the population, exclusions, strata, random seed, and sample size. AI may draft the Python; an approved local environment runs it and produces both the sample and execution log.

CODE 01 · REPRODUCIBLE STRATIFIED SAMPLE
import pandas as pd

population = pd.read_excel("sap_po_population.xlsx")
population["amount"] = pd.to_numeric(
    population["amount"], errors="coerce"
)

sample = (
    population.dropna(subset=["amount"])
    .groupby("business_unit", group_keys=False)
    .sample(n=10, random_state=20260701)
)

sample.to_excel("audit_sample.xlsx", index=False)

Key clue: AI can help write code; auditors still own population completeness and sampling logic. Retain the code version, parameters, input count, and output count. Otherwise, speed merely delivers the mistake earlier.

Case 2: three-way matching—let machines flag disagreement

RPA can retrieve and file documents. OCR can turn scans into text. Rules can compare PO, receipt, and invoice identifiers, values, quantities, and dates. The system flags exceptions; auditors determine whether each one is tolerance, timing, or control failure.

StageSystemAuditor
ExtractOCR and field locationTest recognition accuracy
CompareTolerances and rulesApprove rules and exceptions
ConcludeList differencesVerify cause and evidence

Key clue: text that looks plausible is not necessarily correct OCR. Test against human-labeled samples and force review for low-confidence fields.

Case 3: findings—from blank page to reviewable draft

Provide an approved AI system with de-identified facts, applicable criteria, verified causes, and possible impacts. Ask it to separate known facts, open questions, and recommendations. It may remove blank-page friction; it must never invent interviews or evidence.

A safer prompt frame

“Draft an audit finding using only the material below. Structure it as condition, criteria, cause, impact, and recommendation. Do not add facts. Mark insufficient evidence as [TO VERIFY]. Use neutral language and do not allege fraud, intent, or materiality without support.”

Choose tools for the waters they know

  • Python and pandas: bulk cleaning, sampling, rule tests, and reproducible outputs.
  • Power Automate or UiPath: stable desktop steps, file movement, application interaction, and schedules.
  • OCR: image-to-field extraction, with measured recognition quality.
  • Approved enterprise AI: summarization, classification, drafting, and question generation.
  • Local models: may reduce data exposure, but model provenance, access, logs, and output quality still need governance. Local is not a synonym for safe.

A 30-day closure plan

  1. Week 1 — Record: choose one frequent process and measure five end-to-end runs, including rework.
  2. Week 2 — Decompose: separate judgment, rule execution, data movement, and drafting. Automate only low-risk pieces first.
  3. Week 3 — Pilot: run historical data in parallel and test completeness, accuracy, misses, and traceability.
  4. Week 4 — Decide: compare pilot with baseline. Scale only if it passes; otherwise repair the process and spare the tool the alibi.
CASE CLOSED

Return the recovered hours to work only people can do

Automation takes data preparation, initial matching, and drafting. Auditors reinvest the hours in root-cause analysis, interviews, professional skepticism, and risk communication. The win is not working half as much. It is spending more of the day on work worthy of accountable professionals.

Captain's verdict: baseline first, pilot second. Prove quality did not fall before declaring the hours recovered.
