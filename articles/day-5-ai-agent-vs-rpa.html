AI × AUDIT · DAY 05

AI Agent 是什麼？和 RPA 有何不同？

RPA 像照航線開船的可靠水手；AI Agent 像會看海象、選工具、遇到暗礁就改道的大副。兩者都能幹活，但別把羅盤和船槳叫成同一件事。

Arthur Agent 編輯室2026 年 6 月 30 日約 8 分鐘閱讀
實際案例稽核團隊要用公開資訊初篩漂綠風險：下載年報與永續報告書，比對同年度的排放量、用水量和員工人數，再列出無法勾稽的地方。

流程看似固定，直到第一家公司寫「總用水量」、第二家寫「取水量」，第三家把單位藏在第 87 頁。這時，照表操課的機器人開始冒煙，會判讀情境的 Agent 才剛把帽子戴正。

一句話分辨：RPA 執行你事先畫好的航線；AI Agent 接受目標，根據眼前情況決定下一步，再使用工具完成任務。

先別被新名詞搶走羅盤

RPA（Robotic Process Automation）擅長重複、規則明確、步驟穩定的工作，例如登入系統、下載檔案、搬移資料與填寫欄位。AI Agent 則由模型搭配指令、工具、護欄與執行迴圈組成；它可以根據工具回傳的結果，決定繼續、換方法、要求人工核准或結束。

比較RPAAI Agent
接到什麼固定步驟目標與限制
遇到例外走預設分支或停止判讀情境、改選工具或升級人工
最擅長穩定、重複、可預測非結構資料、多步推理、動態任務
主要風險畫面或規則改了就卡住可能誤判、亂用工具或繞太多圈

同一個漂綠初篩任務，RPA 怎麼做？

先建立公司清單與固定下載網址，再按既定欄位擷取數字、比對差異、輸出 Excel。只要文件格式穩定，它會像守時的水手：不抱怨、不分心，也不會突然研究海怪。

CODE 01 · 固定規則流程
for company in companies:
    annual = download(company.annual_report_url)
    esg = download(company.esg_report_url)

    for metric in ["scope_1", "scope_2", "water", "employees"]:
        left = extract_fixed_field(annual, metric)
        right = extract_fixed_field(esg, metric)

        if left != right:
            write_exception(company, metric, left, right)

這段 code 的優點是可預測、好測試、容易留痕；缺點也很誠實：欄位名稱、頁碼或版型一變，extract_fixed_field() 就可能兩手一攤。

換 AI Agent 上甲板，差別在哪？

Agent 不只收到步驟，而是收到任務：「找出同公司、同年度、同 ESG 指標在公開文件中的重大差異，保留來源與頁碼，不足以判斷時交給人。」接著它重複四件事：觀察、決定、使用工具、檢查結果。

CODE 02 · Agent 概念迴圈
goal = "找出公開 ESG 文件中無法合理勾稽的差異"
state = {"goal": goal, "evidence": [], "turn": 0}

while state["turn"] < 8:
    observation = inspect_documents(state)
    action = planner.choose_next_action(
        observation,
        tools=[search_reports, extract_metric,
               normalize_unit, compare_values]
    )
    result = action.run()
    state["evidence"].append(result)
    state["turn"] += 1

    if action.needs_human_review or evidence_is_enough(state):
        break

export_workpaper(state)

這不是一段可直接連上公司資料的成品，而是 Agent 架構的縮圖：它有目標、有工具、有狀態、有停止條件，也知道何時把舵交回人類。沒有停止條件的 Agent，不叫自主，叫迷航。

漂綠初篩時，Agent 可以多做哪一步？

  1. 辨識相似指標：判斷「總用水量」與「取水量」可能相關，但先標記口徑風險，不擅自當成相同。
  2. 補齊脈絡：尋找單位、年度、揭露邊界、頁碼與第三方確信註記。
  3. 動態選工具：表格可直接解析就讀表格；掃描 PDF 則改走 OCR；缺頁碼就回到原文件搜尋。
  4. 分級例外：四捨五入列低風險；可能是邊界差異列中風險；重大落差且無調節表則升級人工。

雷達可以說前方有東西；船長仍要判斷，那是島、礁石，還是一份把範疇一和範疇二加錯的永續報告書。

所以 Agent 會取代 RPA 嗎？

不必急著把可靠水手丟下船。最實用的組合通常是：Agent 負責讀懂任務、處理例外與決定下一步；RPA 負責登入、下載、搬檔、輸入系統等需要穩定執行的動作；人則負責核准敏感操作與做最終稽核判斷。

規劃與判讀→RPA／工具
可靠執行→稽核人員
覆核與負責

稽核 Agent 上船前，先檢查五件事

  • 任務邊界：哪些網站、檔案與資料可以讀？哪些動作禁止？
  • 工具白名單：只能使用核准工具，寫入、寄信或發布前必須人工核准。
  • 證據可追溯：每個差異都要留下來源文件、頁碼、擷取值與處理步驟。
  • 停止與成本限制：設定最大回合、時間與費用，別讓它追著一張表航行到世界盡頭。
  • 人工最終判斷：Agent 負責初篩；是否構成漂綠、缺失或重大揭露風險，由稽核人員決定。

今天的選型口訣

固定步驟交給 RPA，動態判讀交給 Agent，重大決定留給人。

如果流程每次都一樣，先別為了時髦硬塞 Agent；如果文件格式混亂、例外很多、下一步要看前一步結果，那才是 Agent 值得出航的海域。

案例回航

公開資訊漂綠初篩，最穩的做法不是二選一

RPA 依公司清單定期下載公開文件並命名歸檔；Agent 擷取相似 ESG 指標、統一單位、保留頁碼並把差異分級；稽核人員再檢查邊界、版本與調節表。最後產出的不是「某家公司漂綠」的武斷結論，而是一份知道下一步該追什麼證據的查核清單。
